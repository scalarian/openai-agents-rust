# Visualization

Use this page when you want diagrams or graph-shaped representations of an agent system.

## What Lives Here

Visualization is exposed through the extensions surface. Use it when you want to:

- render agent graphs
- explain a multi-agent topology to teammates
- capture architecture artifacts in docs

## When Visualization Helps

- router and specialist systems
- handoff-heavy workflows
- docs and design reviews
- debugging runtime shape rather than one specific bug

## Practical Rule

Generate diagrams from the real runtime model when possible. Do not maintain a beautiful but stale hand-drawn graph that no longer matches the code.
